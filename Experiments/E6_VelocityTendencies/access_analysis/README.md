# E6 / access_analysis

LOKI-driven analysis of ICON's `velocity_tendencies` subroutine.
Produces the JSON inputs the rest of the E6 pipeline + E8 downstream
consume; see the **Files** table below for what each script writes.

## Files

| File | Kind | Purpose |
|---|---|---|
| `analyze_access.py` | script | LOKI entry. Parses `velocity_advection_preprocessed.f90` → `*.analysis.md`. |
| `select_loopnests.py` | script | Picks 6 representative nests by pattern class + tiebreaker (loopnest_1 manually pinned). → `layout_candidates.json` + `chosen_loopnests.md`. |
| `generate_canonical_groups.py` | script | Derives the 6-group layout taxonomy (cv / ch / f / s / n / lm) from `layout_candidates.json`. Connectivity aliases are the one piece LOKI can't expose — they're hardcoded in this script. |
| `find_groups_and_loopnests.py` | script | Reporter. Prints the 6 access groups + the 6 chosen loopnests side-by-side and writes `groups_and_loopnests.json` (read-only summary). |
| `derive_winners.py` | script | Reads `../loopnest_{1..6}/results/{platform}/*.csv` (filtered to `cell_dist=exact` = R02B05 connectivity), picks the lowest-time non-blocked V_k per (loopnest, platform, backend), and emits `winners.json` with per-target winners + cross-product axes. **Consumed by E8's `run_{daint,beverin}.sh`** (auto-regenerated on first submission if missing). |
| `generate_layout_groups.py` | script | Self-contained spec emitter. Encodes paper §IV-A's "7 layout-equivalent groups + V_k permutation per group" into `layout_groups.json` (audit-trail JSON for the AD). **Consumed by E8** to build `winner_v1`/`winner_v2`/`winner_v6` named configs. |
| `velocity_advection_preprocessed.f90` | input | Fortran source analyzed. Preprocessed so `#include` / `!$OMP` are already expanded. |
| `velocity_advection_preprocessed.analysis.md` | output | Auto-generated by `analyze_access.py`. 25 nests + array groups + per-array summaries. |
| `layout_candidates.json` | output | Structured record of the 6 chosen nests and candidates per class. Consumed by `../generate_winners.py`, `../build_layout_crossproduct.py`, and `../../E8_LegacyVT/run_stage8_permutations.py`. |
| `chosen_loopnests.md` | output | Human-readable mirror of `layout_candidates.json`. |
| `canonical_array_groups.json` | output | Auto-generated 6-group taxonomy. Every group is also tagged with its storage axis (`IC` for cv/ch/f/s/lm; `IN` for n). Consumed by `../generate_winners.py` and `../build_layout_crossproduct.py`. |
| `groups_and_loopnests.json` | output | Auto-generated by `find_groups_and_loopnests.py`. Single-file mirror of the 6 groups + 6 loopnests for downstream use. |
| `winners.json` | output | Auto-generated by `derive_winners.py`. Per-(loopnest, platform, backend) empirical V_k winners + cross-product spec. The auditable bridge between E6 (per-loopnest measurements) and E8 (full-VT sweep). |
| `layout_groups.json` | output | Auto-generated by `generate_layout_groups.py`. The 7 layout-equivalent groups, their array members, and the V1/V2/V6 permutation per group. The auditable spec for E8's named V_k winners. |

## Running

```bash
source ../../common/activate.sh   # venv with LOKI installed

python3 analyze_access.py velocity_advection_preprocessed.f90
# -> velocity_advection_preprocessed.analysis.md

python3 select_loopnests.py
# -> layout_candidates.json, chosen_loopnests.md

python3 generate_canonical_groups.py
# -> canonical_array_groups.json

python3 find_groups_and_loopnests.py
# -> groups_and_loopnests.json (and a printed summary)

python3 derive_winners.py
# -> winners.json (per-loopnest V_k winners; needs loopnest_*/results/*.csv)

python3 generate_layout_groups.py
# -> layout_groups.json (paper-§IV-A 7-group V_k spec, no inputs)
```

`winners.json` and `layout_groups.json` are **auto-regenerated by E8**'s
`run_{daint,beverin}.sh` if absent at submission time, so the manual
invocation above is only needed when the upstream loopnest CSVs change
or you want to inspect the audit trail without running E8.

## How the 6 canonical groups are derived

`generate_canonical_groups.py` walks every (array, touching-nest-set)
pair in `layout_candidates.json` and assigns it one of six tags. Each
group also carries a `group_axis` consumed by the cross-product
builder (`IC` = h_first vs v_first choice; `IN` = SoA vs AoS choice).

| Group | Axis | Rule |
|---|---|---|
| `n` — CONN | IN | Name matches `p_patch%{cells\|edges\|verts}%(edge\|cell\|vertex\|neighbor\|quad)_(idx\|blk)`, OR a local alias (`icidx`, `icblk`, `ieidx`, `ieblk`, `ividx`, `ivblk`, `iqidx`, `iqblk`, `incidx`, `incblk`). |
| `f` — FIELDS | IC | Name begins with `p_prog%` or `p_diag%`. |
| `s` — STENCIL | IC | Name begins with `p_metrics%`, `p_int%`, or `p_patch%{edges,cells}%*` (non-connectivity). |
| `cv` — COMPUTE_VERT | IC | Work array (`z_*`, `cfl_*`) that appears in at least one nest with shape `v.h` AND `full_vert` range. |
| `ch` — COMPUTE_HORIZ | IC | Work array that doesn't meet the `cv` criterion. |
| `lm` — LEVMASK | IC | Level-mask 2-D arrays (`levmask`, `levelmask`) — split out from `cv` so the `(nblks, nlev) → (nlev, nblks)` transpose can be expressed independently. |

Everything except the connectivity aliases is derived from LOKI
output. The alias list is hardcoded because `analyze_access.py`
filters indirection arrays out of every nest's `arrays` field, so
they need re-injection here.

The cross-product builder uses `group_axis` to project each per-group
V-winner onto the right axis before voting. V-ids that differ only in
the opposite axis (e.g. V3 vs V4 on IC, or V1 vs V3 on IN) collapse
to the same candidate, keeping the final sweep small.

**Caveat.** The `cv` / `ch` boundary in the real sweep
(`sc26_layout/permute_stage8.py`) is drawn by the *write direction*
of each producer loop (jk-outer write → cv; je-inner write → ch).
LOKI's output doesn't split reads from writes, so this script falls
back to "touching nest shape" — expect small differences (e.g.
`z_v_grad_w` lands in `cv` here but in `ch` in the reference repo).
