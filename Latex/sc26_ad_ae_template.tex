\section{Overview of Contributions and Artifacts}

\subsection{Paper's Main Contributions}

\begin{description}
\item[$C_1$] A theoretical proof, via blocked CDAGs and the Red-Blue-White Pebble Game, that a poor data layout can inflate the I/O lower bound by up to a factor of the cache-line length $B$ relative to a layout-aware optimum, and that no schedule can close this gap. The full proof, cut from the paper for space, is bundled with the artifact at \texttt{Proofs/necessity\_proof.pdf}; it is not exercised by the software.
\item[$C_2$] A composable layout algebra of five primitives (pad, permute, block, shuffle, zip) whose compositions express any bijective array rearrangement.
\item[$C_3$] Two heuristic cost metrics --- the average new-block count $\mu$ (guiding permutation) and the average block distance $\Delta$ (guiding blocking) --- with a cookbook (Table~II) mapping observable performance symptoms to corrective primitives.
\item[$C_4$] Empirical validation on microbenchmarks and two production HPC codes (Quantum ESPRESSO, ICON dynamical core) showing up to $1.18\times$ speedup on GPUs and $1.12\times$ on CPUs.
\end{description}

\subsection{Computational Artifacts}

\begin{description}
\item[$A_1$] \url{https://doi.org/10.5281/zenodo.19827100} (concept DOI; resolves to the latest archived version). Source repository: \url{https://github.com/ThrudPrimrose/SC26-Layout-AD}.
\end{description}

\paragraph{E8 is the AD's default full velocity-tendencies path.}
\texttt{Experiments/E8\_LegacyVT/} drives the stage-8 codegen pipeline on DaCe \texttt{f2dace/staging} and reproduces Fig.~14 / Tab.~V from a single \texttt{sbatch run\_\{daint,beverin\}.sh}. The curated default sweep is 67 configs / 134 binaries: the three named ablations (\texttt{nlev\_first}, \texttt{index\_only}, \texttt{winner\_v1} = unpermuted) plus 64 unique \texttt{v123\_*} cells from the per-group V$_k$ cross-product (E6's \texttt{layout\_crossproduct\_v123.json}). The named winners V1 (identity, Fig.~13 \emph{Original Layout}), V2 (connectivity-only intermediate), and V6 ($v_\textrm{first}$, Fig.~13 \emph{Optimized Layout}) appear among the v123 cells; pin them via \texttt{CONFIGS="winner\_v1,winner\_v2,winner\_v6"} for the minimal Fig.~14 reproduction. \texttt{E7\_FullVelocityTendencies/} is an in-progress SDFG-driven refactor of the same pipeline (DaCe \texttt{yakup/dev}) and is not required for the AD's reproduction path.

\newartifact

\artrel

Artifact~$A_1$ validates the layout algebra ($C_2$), the cost metrics ($C_3$), and the cookbook-driven workflow ($C_4$). E1--E4 are microbenchmarks that isolate individual primitives. E5 evaluates composed transformations on a Quantum ESPRESSO production kernel. E6 applies the full optimization workflow per loop-nest pattern class. E8 drives the resulting layouts through the velocity-tendencies module end-to-end on the GPU.

\artexp

Each experiment maps to one paper figure or table (see the experiments table above). Reviewers should observe the following qualitative orderings on any hardware with comparable cache-line sizes; absolute speedups depend on the platform.

\begin{itemize}\itemsep1pt
\item \textbf{E1 (Fig.~4):} permuted $>$ schedule-only on every backend; GPUs reach near STREAM peak.
\item \textbf{E2 (Fig.~8):} AoSoA-16 robust across all four backends; SoA degrades on Zen4 at high $P$ from L1D conflicts.
\item \textbf{E3 (Fig.~9, Tab.~III):} blocked $>$ HPTT $>$ naive row-major on CPUs; GPU layouts converge.
\item \textbf{E4 (Fig.~10):} index-sort $>$ unsorted on every backend; gap largest on GPUs.
\item \textbf{E5 (Fig.~11):} composed unzip+permute+shuffle $\geq$ baseline; gain largest on Hopper.
\item \textbf{E6 (Figs.~12--13, Tab.~IV):} a V$_k$ winner beats the ICON-default per loopnest; the indirect-stencil loopnest shows the largest gain under uniform random indirection.
\item \textbf{E8 (Fig.~14, Tab.~V):} V6 (vertical-first) $\geq$ V1, V2, and unpermuted on Config~B; gain largest on MI300A.
\end{itemize}

\paragraph{Reproducibility caveat.} \%-of-STREAM annotations may differ slightly from the paper's: the cache-flush kernel was rewritten as a 3-sweep $8192^{2}$ Jacobi with buffer swap to defeat dead-code-elimination by the GPU compiler, the MI300A STREAM Triad peak was re-measured against the corrected flush, and every figure in this artifact is normalized to that peak. Qualitative trends --- layout orderings, gap between \texttt{unpermuted} and the V$_k$ winners --- are unchanged.

\artin

\artinpart{Hardware}

Two HPC clusters at the Swiss National Supercomputing Centre (CSCS), accessed via SLURM with exclusive single-node allocations:

\begin{itemize}
\item \textbf{Daint.Alps:} Quad-GH200 nodes, 72-core Arm Neoverse~V2 Grace (512\,GB/s LPDDR5X) + NVIDIA H200 (4\,TB/s HBM3). Measured STREAM Triad peaks: CPU 1.61\,TB/s, GPU 3.78\,TB/s.
\item \textbf{Beverin:} Quad-MI300A APU nodes, 24 AMD Zen\,4 cores sharing 128\,GB unified HBM3. Measured STREAM Triad peaks: CPU 0.97\,TB/s, GPU 3.55\,TB/s.
\end{itemize}

Results should generalize to any system with comparable cache-line sizes (64\,B CPU, 128\,B GPU); absolute bandwidths will differ.

\paragraph{CUDA pin.} \textbf{CUDA must be 12.x} on Daint: CUDA~13 removes or relocates symbols DaCe depends on (\texttt{CTA\_SYNC}, \texttt{cub::CountingInputIterator}, \texttt{cub::TransformInputIterator}, plus \texttt{cudaDeviceProp} field renames). The cluster pin via \texttt{Experiments/common/setup\_daint.sh} (\texttt{cuda@12.9}) avoids them; CUDA~12.9 is verified end-to-end.\par
E8's \texttt{tools/patch\_codegen\_reductions.py} is wired into \texttt{compile\_action()} as a post-codegen hook: between codegen and \texttt{nvcc}/\texttt{hipcc} it restores \texttt{reductions\_device.cuh} into every device-scope file (the permute pass drops it from \texttt{sdfg.global\_code}) so reduction tasklets inlined into \texttt{\_\_device\_\_} helpers compile cleanly. Idempotent.

\paragraph{Compile flags and pinning.} \texttt{CPU\_CXXFLAGS} and \texttt{GPU\_CXXFLAGS} (exported by the setup scripts, applied uniformly across all \texttt{g++} / \texttt{nvcc} / \texttt{hipcc} invocations) carry \texttt{-O3 -march=native -ffast-math -fopenmp -std=c++17} on both platforms; \texttt{-arch=sm\_90 --use\_fast\_math} on Daint; \texttt{--offload-arch=gfx942} with HIP-specific atomics on Beverin. \texttt{OMP\_SCHEDULE=static} and \texttt{SLURM\_CPU\_BIND=cores} are exported alongside; every \texttt{\#pragma omp parallel for} uses \texttt{schedule(static)}. Exact lists in each \texttt{setup\_<plat>.sh}.

\artinpart{Datasets / Inputs}

E1--E4 generate input data synthetically. Production-code experiments (E5, E6, E8) auto-fetch their datasets on first \texttt{sbatch} via per-experiment \texttt{download\_data.sh} (idempotent guard):

\begin{itemize}
\item \textbf{E5:} BaTiO$_3$ from a QE \texttt{pw.x} HSE06 self-consistent calculation (10-atom cell), fetched from ETH PolyBox.
\item \textbf{E6:} R02B05 ICON dataset, with the \texttt{nblks} and \texttt{nproma} dimensions collapsed into a single horizontal axis of 81\,920 cells $\times$ 90 vertical levels (per-loopnest sweeps work on the flattened layout).
\item \textbf{E8:} R02B05 ICON dataset blocked at \texttt{nproma=20480} ($\sim$9~GB; 4 blocks of 20\,480 cells each, 90 vertical levels), shared with E7 via \texttt{LOCAL\_DATA\_DIR} (directory name is historical).
\end{itemize}

\artinpart{Installation and Deployment}

One-time per cluster: \texttt{bash Experiments/common/setup.sh} bootstraps a Python 3.13 venv, the DaCe checkout, and the per-platform compile flags. Per experiment: \texttt{cd Experiments/EX\_\textit{name} \&\& sbatch run\_<plat>.sh}, then plot with \texttt{python plot\_paper.py} (or \texttt{bash Figures/plot\_all.sh Runtime} to regenerate the full runtime sweep). Each \texttt{run\_<plat>.sh} sources \texttt{../common/activate.sh} (venv + DaCe pin) and \texttt{../common/setup\_<plat>.sh} (Spack loads, compile flags, OMP and NUMA pinning) before invoking the benchmark; \texttt{<plat>} is \texttt{daint} or \texttt{beverin}.

\artcomp

Every experiment is driven by its \texttt{run\_<plat>.sh}, which compiles the benchmark, runs 100 timed repetitions after 5 warm-ups, and writes results next to the experiment. Plotting is invoked manually with \texttt{python plot\_paper.py} once the job completes. E1--E5 are mutually independent and may be submitted in parallel; E6 and E8 are fully automated by their run scripts, so reviewers do not invoke any intermediate stage by hand.

\paragraph{Per-experiment workflow.}
\begin{itemize}
\item \textbf{E1} (Fig.~4) --- \texttt{E1\_MatrixAdd/run\_<plat>.sh} compiles \texttt{bench\_\{cpu,gpu\}}, runs the schedule-only vs.\ permuted sweep; \texttt{plot\_paper.py} emits Fig.~4.
\item \textbf{E2} (Fig.~8) --- \texttt{E2\_Conjugation/run\_<plat>.sh} sweeps $P$ over AoS / SoA / AoSoA-16; \texttt{plot\_paper.py} emits Fig.~8.
\item \textbf{E3} (Fig.~9, Tab.~III) --- \texttt{E3\_Transpose/run\_<plat>.sh} runs the transpose sweep against HPTT / cuTENSOR / hipTensor and a follow-up \texttt{cost\_metrics} pass for $\mu$ and $\Delta$; \texttt{plot\_paper.py} emits Fig.~9 and Tab.~III.
\item \textbf{E4} (Fig.~10) --- \texttt{E4\_GAS/run\_<plat>.sh} auto-fetches BaTiO$_3$ if absent, runs uniform-random and index-sort gather--accumulate--scatter; \texttt{plot\_paper.py} emits Fig.~10.
\item \textbf{E5} (Fig.~11) --- \texttt{E5\_USXX/run\_<plat>.sh} auto-fetches BaTiO$_3$ HSE06 state if absent, runs the composed unzip + permute + shuffle on \texttt{addusxx\_g}; \texttt{plot\_paper.py} emits Fig.~11.
\item \textbf{E6} (Figs.~12--13, Tab.~IV) --- each \texttt{loopnest\_\{1..6\}/run\_<plat>.sh} (under \texttt{E6\_VelocityTendencies/}) automates the full per-loopnest pipeline:
  \begin{itemize}\itemsep1pt
  \item $T_{6.1}$ data fetch (R02B05 ICON state).
  \item $T_{6.2}$ LOKI access analysis $\to$ \texttt{layout\_candidates.json}.
  \item $T_{6.3}$ 6-group taxonomy (\texttt{cv}, \texttt{ch}, \texttt{f}, \texttt{s}, \texttt{n}, \texttt{lm}) $\to$ \texttt{canonical\_array\_groups.json}.
  \item $T_{6.4}$ layout sweep over $nlev \in \{90, 96, 128, 256\}$, default 128.
  \item $T_{6.5}$ winners cross-product (\texttt{generate\_winners.py} / \texttt{derive\_winners.py} / \texttt{generate\_layout\_groups.py}) $\to$ \texttt{winners.json}, \texttt{layout\_groups.json}.
  \item $T_{6.6}$ \texttt{plot\_paper.py} $\to$ Figs.~12--13, Tab.~IV.
  \end{itemize}
  \texttt{winners.json} feeds E8.
\item \textbf{E8} (Fig.~14, Tab.~V) --- \texttt{E8\_LegacyVT/run\_<plat>.sh} automates:
  \begin{itemize}\itemsep1pt
  \item $T_{8.1}$ dataset resolution (existing E8 dir $\to$ E7 symlink $\to$ fresh fetch).
  \item $T_{8.2}$ regenerate \texttt{winners.json} / \texttt{layout\_candidates.json} if missing.
  \item $T_{8.3}$ per-config compile via \texttt{compile\_gpu\_stage8}.
  \item $T_{8.4}$ run \texttt{run\_stage8\_permutations.py} for each \texttt{CONFIGS} entry (default \texttt{winner\_v\{1,2,6\}}); stdout per (config $\times$ shuffle $\times$ timestep) lands in \texttt{<plat>\_full\_permutations\_8/} as one TXT each.
  \item $T_{8.5}$ plot via \texttt{plot\_paper\_snapshot.sh} or \texttt{plot\_results.sh} (under \texttt{Figures/}).
  \end{itemize}
\end{itemize}

\paragraph{Statistical methodology.} Following Hoefler and Belli, every configuration runs 100 repetitions after 5 warm-ups, with cache flushes between repetitions (an $8192^2$ Jacobi stencil over 3 sweeps with buffer swap; \texttt{common/jacobi\_flush.h}). We report medians with 95\% BCa-bootstrap confidence intervals ($n=10{,}000$ resamples). Violin plots show all 100 samples without outlier trimming; every violin uses the same KDE settings (Scott's rule, 200 evaluation points). The RNG seed is \texttt{SC26\_SEED=42}, and bandwidths are normalized to the measured STREAM Triad peak per platform.
